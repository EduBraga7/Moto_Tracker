# Regras do Projeto: Moto Tracker

## 1. Idioma e Ortografia (Estritamente pt-BR)
- Todo texto visível para o usuário (HTML, respostas do Telegram, mensagens de erro, logs) DEVE ser escrito em Português do Brasil (pt-BR) impecável.
- É OBRIGATÓRIO o uso correto de acentos (á, é, í, ó, ú, ã, õ, ç). NUNCA gere palavras como "Saude", "Mes", "Media", "Pecas", "Oleo".
- Sempre utilize `<meta charset="UTF-8">` no `<head>` dos arquivos HTML.

## 2. Padrão de Nomenclatura (Vocabulário da Moto)
- Use "KM" (maiúsculo) para quilometragem.
- Use "Custo/KM" e "km/L".
- Peças padrão: "Óleo", "Relação", "Pneus", "Filtro de Ar".
- Nunca chame a moto de "carro" ou "veículo" genérico. É uma Honda NXR 150 Bros, trate o contexto como tal.

## 3. Arquitetura e Código
- Stack atual: Python (Flask), Supabase/Firebase, Vercel Serverless.
- Estilo de Design: "Moto-Noir Digital" (Dark mode, glassmorphism, glowing accents em ciano e laranja).
- NUNCA use `except: pass`. Sempre faça o log do erro adequadamente.
- As rotas do Flask devem ser enxutas, chamando regras de negócio isoladas no `services.py`.
