================================================================
PROJETO: MOTO TRACKER (SaaS & Minimalist Edition)
================================================================
OBJETIVO: 
Plataforma minimalista em formato PWA para gestão inteligente 
de consumo e manutenção de veículos. Nascido para uma Bros 150, 
mas arquitetado para escalar para qualquer veículo.

----------------------------------------------------------------
1. FUNCIONALIDADES PRINCIPAIS (O "Core" do App)
----------------------------------------------------------------

[A] MÓDULO ABASTECIMENTO (O Fluxo das 3 Caixinhas)
   - Input Principal: 
     1. KM Total do Painel (Evita erro de Trip zerado).
     2. Litros Abastecidos.
     3. Valor Total Pago (R$).
   - Input Secundário (Apoio): "Abastecimento Rápido" (Apenas valor em R$ para emergências).
   - Output Automático (Mágica do Código): 
     * Média de Consumo (KM/L) calculada pela diferença do último KM.
     * Custo por KM Rodado (R$/KM).
     * Preço exato do Litro (Valor / Litros).

[B] MÓDULO GARAGEM & SAÚDE (Manutenção Dinâmica)
   - Input de Peças: O usuário cria o item (ex: "Kit Relação", "Óleo Motul") e define o intervalo de troca (ex: 15.000km).
   - Output: Barra de progresso visual (ex: 80% de vida útil) indicando o desgaste com base no KM atual do veículo.
   - Histórico: Registro de valores gastos por peça/oficina.

[C] EXPERIÊNCIA DO USUÁRIO (UX Premium)
   - Saudação inteligente baseada no horário ("Bom dia, Eduardo!").
   - Dashboard limpo focado no veículo selecionado.
   - Tema Dark / Glassmorphism para conforto visual na rua.

----------------------------------------------------------------
2. ESTRUTURA DE DADOS (Arquitetura Escalável e LGPD)
----------------------------------------------------------------
A estrutura abandona o relacional simples (SQLite) e vai para NoSQL (Firebase Firestore) em formato de Árvore, garantindo segurança (cada usuário só vê seus dados) e anonimato (sem placas de veículos).

COLEÇÃO: usuarios
  -> documento: user_id (Gerado pelo Google Auth)
     - email
     - nome
     -> SUB-COLEÇÃO: veiculos
        -> documento: veiculo_id
           - apelido (ex: "Bros 150 - Dia a dia")
           - tipo (Moto/Carro)
           - km_inicial
           -> SUB-COLEÇÃO: abastecimentos (km, litros, valor, data)
           -> SUB-COLEÇÃO: manutencoes (km, peca, valor, data)
           -> SUB-COLEÇÃO: pecas_rastreadas (nome_peca, intervalo_troca)

----------------------------------------------------------------
3. TECNOLOGIAS (Stack de Alta Performance)
----------------------------------------------------------------
- Backend & Regras de Negócio: Python (Flask) ou Serverless.
- Frontend & UX: HTML/CSS/JS (Foco em Glassmorphism e PWA).
- Banco de Dados & Auth: Google Firebase (Firestore + Autenticação).
- Hospedagem: Vercel (Deploy contínuo e rápido).
